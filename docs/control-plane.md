# AI Control Plane

## Purpose

Host long-lived platform services for the local AI environment.

## Phase 1

- Docker
- Docker Compose
- Open WebUI

## Phase 2

- LiteLLM
- GitHub MCP Server

## Phase 3

- Observability Dashboard
- Retro Service
- Agent Runtime Services

## Requirements

- Reproducible from repository automation
- Persistent data volumes
- Accessible from local network
- Deployable to a fresh Ubuntu VM

## Success Criteria

A fresh VM can be provisioned from repository automation and Open WebUI is accessible from the local network.
